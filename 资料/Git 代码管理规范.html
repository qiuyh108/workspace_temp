Git代码管理规范

1：代码库分支管理

远端分支由3部分构成：

  • 主线分支（ master）
  • 开发分支（ dev）
  • 标签分支（ tag)
dev 开发中的任务只能在这个分支进行。
master 分支是所有已上线功能代码的集合。
tag 分支是每一个线上版本对应的节点代码。

本地分支：

  • 关联远端的本地分支 （dev）
  • 本地个人分支 ( dev_xxx)

2：代码获取和分支切换

  • 克隆dev分支， 其中 xxxDev 为开发分支名, repo 为代码的仓库地址。

    git clone -b dev repo

  • 创建本地个人开发分支dev_xxx, 并切换至 dev

    git checkout -b dev

  • 日常开发在 dev_xxx 分支进行

  • 所有分支可通过 git branch -a 查看

3：拉取合并远端代码到本地

  • 步骤一：git add . 将当前目录的所有更改文件添加到暂存区

  • 步骤二：git commit -m “xxx”, 将暂存区的文件提交到本地仓库

  • 步骤三：git checkout dev 从 dev_ xxx 分支切换到 dev 分支。

  • 步骤四：git pull 同步 dev分支的远端代码。

  • 步骤五：git checkout dev_xxx 再从 dev 分支切换回个人分支 dev_ xxx。

  • 步骤六：git rebase dev 在个人分支dev_ xxx 使用, 完成代码的合并。

4: 将自己的代码合并到远端

  • 先执行完 3 保证没有冲突、运行不出错。如果有冲突 使用 git status 查看冲突文件。

  • 有冲突或者错误先改正，改正完后使用 git add . 保存修改，然后用 git rebase --continue 完成合并。

  • 程序能正常运行使用 git push 将dev_xxx分支推送到远端，创建 merge request，将代码合并到dev分支。
